Train Tracked

Created as a Level 5 project @UoP, A Better™ UK train tracking app which uses National Rail Enquries' Darwin™ API. Built with Flutter on the frontend, Dart & Shelf on the backend.

The backend acts as a proxy between the mobile app and Darwin. This is mostly to convert from the legacy SOAP API that is Darwin and a more up-to-date and reliable REST API. It also improves the reliability of requests as SOAP relies heavily on keeping a connection open for the duration of the request, including while processing in the background, which is not always feasable on the type of connection people may be using on a train (i.e. cellular data). There is also more filtering done before passing the request to Darwin, as it has no inate validation (doesn't check dates, times or that a train RID or station CRS exists, simply returning a 500...) so that less time is wasted waiting for it to respond with a useless error.
